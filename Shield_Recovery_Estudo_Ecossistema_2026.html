O projeto evoluiu de demo para plataforma operacional

O Shield Recovery nao e mais apenas um CRM de leads. Evoluiu para um ecossistema de recuperacao de pagamentos com 3 camadas integradas: Recovery Engine, Checkout Platform e AI Followup. A maioria dos componentes criticos agora esta funcional e pronta para producao.

A tese mudou

Antes era: "plataforma de CRM com IA para recovery".

Agora e: "ecossistema completo de recuperacao de pagamentos que vai do webhook ate o checkout, com IA decidindo cada passo".

A separacao do Checkout em plataforma standalone e o movimento mais importante. Transforma o Shield de "ferramenta de recovery" em infraestrutura de pagamentos.

Impacto: Agora o Shield nao depende do gateway original para gerar links de pagamento. Ele cria seus proprios checkouts.

ECOSSISTEMA
3 plataformas, 1 fluxo unico

O Shield Recovery agora opera como um ecossistema de 3 camadas que se comunicam via webhooks e APIs internas.

CAMADA 1

Shield Recovery

Motor principal. Recebe webhooks, cria leads, classifica com IA, decide estrategia, agenda automacoes, envia mensagens, gerencia conversas.

Stack: Next.js + Supabase + Vercel

~15.000 linhas de logica server-side

CAMADA 2

Checkout Platform

Plataforma standalone de checkout. Cria sessoes de pagamento, hospeda pagina de checkout, processa via PagNet/Stripe, retorna resultado.

Stack: Next.js separado

API: POST /api/v1/sessions

CAMADA 3

AI Followup Engine

Motor de decisao que classifica intents, decide proxima acao, gera mensagens personalizadas e orquestra o ciclo de follow-up multicanal.

Core: Orchestrator (814 linhas)

Templates: 61 mensagens em 3 canais

04
INTELIGENCIA ARTIFICIAL
O motor de decisao mais sofisticado do sistema

O AI Orchestrator evoluiu de um agregador de dados para um sistema de decisao completo com 814 linhas que classifica, decide e orquestra cada interacao de recovery.

decideRecoveryPlan()

Input: contact + payment + conversation + automation policy

Output (12 campos):

  • classification — score 0-100 + probability
  • strategy — matched playbook
  • nextAction — acao concreta a executar
  • urgency — immediate / today / scheduled
  • channel — whatsapp / email
  • tone — direct / empathetic / urgent / casual
  • followUpMode — autonomous / supervised / manual
  • requiresHuman — boolean
  • shouldGeneratePaymentLink — boolean
  • timingMinutes — delay antes de enviar

classifyInboundIntent()

Input: mensagem de texto do cliente

12 intents detectados:

  • payment_method_pix — "1", "pix", "quero pix" (0.95)
  • payment_method_card — "2", "cartao", "credito" (0.95)
  • payment_method_boleto — "3", "boleto" (0.95)
  • payment_intent — "vou pagar", "manda o link"
  • human_handoff — "quero falar com humano"
  • friction — "pare", "reclam", "raiva"
  • needs_time — "depois", "amanha"
  • objection — "nao consigo", "erro", "caro"
  • question — qualquer "?"
  • confirmation — "ok", "sim", "pode ser"
  • irrelevant — default
A diferenca fundamental: A IA nao e mais cosmetic. Cada mensagem do cliente passa por classifyInboundIntent(), que alimenta decideConversationFollowUp(), que decide a proxima acao com contexto completo. O operador pode atuar em modo autonomo, supervisionado ou manual. Tudo com audit trail.
05
CHECKOUT FLOW
Do WhatsApp ao pagamento em 4 passos

O fluxo mais critico do sistema: como o cliente vai de "pagamento falhou" para "pagamento concluido".

1

Shield envia mensagem com selecao de metodo

Apos webhook de falha, AI decide nextAction: "ask_payment_method". Mensagem enviada via WhatsApp:

"Oi {nome}, vi que houve um problema no seu pagamento de R${valor}. Posso te ajudar! Como prefere pagar? 1 - PIX / 2 - Cartao / 3 - Boleto"

2

Cliente responde com metodo escolhido

Cliente digita "1" ou "pix". WhatsApp webhook recebe a mensagem.

classifyInboundIntent("1") retorna intent: "payment_method_pix" com confidence: 0.95

decideConversationFollowUp() retorna nextAction: "generate_method_payment_link"

3

Shield gera sessao de checkout com metodo pre-selecionado

createCheckoutSession() chamado na Checkout Platform standalone:

Amount: R${valor} | Method: pix | Customer: {nome, email, phone, doc}

Retorna: checkoutUrl + ?method=pix

Cria PaymentAttempt no banco para tracking.

4

Mensagem com link + PIX code enviada ao cliente

AI gera mensagem personalizada:

"Perfeito! Gerando seu pagamento via PIX. Codigo Pix copia e cola: {pixCode}. Ou clique aqui: {checkoutUrl}"

Cliente paga na plataforma de checkout → webhook payment_succeeded → Lead marcado RECOVERED

O ciclo completo funciona end-to-end. Da falha de pagamento ate a confirmacao da recuperacao, passando por selecao de metodo via WhatsApp e checkout em plataforma separada. Todo passo e rastreado e auditavel.

Autonomy Modes

ModoComportamento
AutonomousIA responde e gera links sem aprovacao. Seller acompanha.
SupervisedIA sugere acao, seller aprova antes de enviar.
ManualSeller opera sozinho. IA so classifica e recomenda.

Admin define por seller: automationsEnabled, inboxEnabled, autonomyMode, maxAssignedLeads, recoveryTargetPercent.

Seller-scoped webhooks

Cada seller tem URL propria: /api/webhooks/shield-gateway/{sellerKey}

GAPS
O que ainda falta para 10/10

Retry Page (handoff)

A pagina /retry/[id] e um placeholder. Precisa redirecionar para a Checkout Platform com a sessao correta. Esforco: 1 dia.

WhatsApp Interactive Buttons

Sistema pronto para button_reply no webhook parser, mas envio usa texto numerado (1/2/3). Migrar para botoes nativos melhora UX. Esforco: 2 dias.

Boleto flow completo

Metadata de boleto e capturada na normalizacao, mas nao ha flow especifico de envio/lembrete de boleto. Esforco: 2-3 dias.

Analytics avancado

Metricas basicas existem. Falta: funnel por estrategia, tempo medio por etapa, heatmap de horarios, A/B testing de mensagens. Esforco: 5 dias.

Prisma cleanup

schema.prisma ainda existe mas nao e usado. Limpar. 1h.

Cron automatico

Worker roda via endpoint. Configurar Vercel Cron para executar a cada 1-5 min. 30min.

Total estimado para 10/10: 10-12 dias uteis. Nenhum gap e bloqueante. O sistema ja opera end-to-end. Esses sao refinamentos, nao fundacoes.

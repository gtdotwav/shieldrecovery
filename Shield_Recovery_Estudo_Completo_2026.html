INTELIGENCIA ARTIFICIAL
A verdade sobre a camada de IA

A IA e o diferencial narrativo do Shield Recovery. Mas hoje, nenhuma chamada real a qualquer LLM acontece. O campo openAiApiKey existe no config mas nunca e usado.

Recovery Classifier

Status: Deterministico

Score 0-100 calculado por regras fixas: tem telefone? (+20), tem email? (+15), valor > R$100? (+10), falha recente? (+15). Nao usa ML nem LLM.

Veredicto: Funcional para MVP. Substituir por modelo treinado na Fase 4.

Message Generator

Status: Template puro

Comentario literal no codigo: "In production this would call an LLM... For now we use high-quality handcrafted templates." Substituicao de variaveis {name}, {value}, {link}.

Veredicto: Templates sao bons. Mas o marketing diz "IA gera mensagens" — isso e falso hoje.

Strategy Engine

Status: Hardcoded

5 playbooks definidos como constantes TypeScript. Condicoes (user_responded, payment_confirmed) sao strings descritivas — nao sao avaliadas pelo worker.

Veredicto: Estrutura correta. Precisa migrar para DB e avaliar condicoes em runtime.

AI Orchestrator

Status: Agregador

Agrega dados do classifier + strategies + metricas para o dashboard /ai. Funciona como agregador de dados, nao como orquestrador inteligente.

Veredicto: Renomear para "Recovery Analytics Aggregator" ate que IA real exista.

Recomendacao arquitetural

Implementar OpenAI na generateRecoveryMessage() e generateConversationReply() requer ~50 linhas de codigo. O campo openAiApiKey ja existe no config. A decisao e de produto, nao de engenharia. Custo estimado: ~$0.002 por mensagem com GPT-4o-mini.

04
GAPS CRITICOS
O que impede o produto de operar

Tres bloqueios estruturais impedem o Shield Recovery de funcionar como produto real em producao.

1. Nenhuma mensagem sai do sistema

O MessagingService.dispatchOutboundMessage() existe como interface mas o envio real via WhatsApp Cloud API nao esta conectado. Mensagens sao criadas no banco com status "queued" mas nunca entregues ao destinatario.

Impacto: O pipeline inteiro (webhook → lead → mensagem) funciona ate o penultimo passo. O ultimo passo (enviar) nao acontece.

Esforco para resolver: 2-3 dias. Integrar WhatsApp Cloud API send endpoint + SendGrid email API.

2. Jobs agendados nunca executam

O RecoveryAutomationService cria QueueJobRecord no banco com runAt futuro, mas nao existe BullMQ, Redis nem worker runner para processar esses jobs. O package.json nao inclui nenhuma dessas dependencias.

Impacto: Toda a automacao (cadencia de follow-up, retries, escalacoes) e teatro. Jobs existem no DB mas ninguem os executa.

Esforco para resolver: 5-7 dias. Instalar BullMQ + Redis, criar worker que poll jobs e executa steps.

3. Persistencia com caminho paralelo morto

Supabase funciona. Mas schema.prisma existe no projeto, Prisma nao esta no package.json, e o .env.example referencia DATABASE_URL que nao e usada. Isso confunde qualquer dev novo.

Impacto: Confusao, nao bloqueio. Mas se nao limpar agora, vai acumular.

Esforco para resolver: 1 dia. Deletar schema.prisma, limpar envs, documentar que Supabase e a fonte unica.

ALAVANCAS
7 alavancas de receita e recorrencia

1. Success Fee escalonado

Quanto mais recupera, menor o %. Incentiva upgrade de plano para reduzir taxa. Cliente que recupera R$50k/mes no Starter (8% = R$4k) migra para Enterprise (3% = R$1.5k + R$1.497 fixo = R$2.997). Economia de R$1k/mes incentiva upgrade.

2. Creditos de IA por uso

Cada mensagem gerada por LLM custa ~R$0.01 (GPT-4o-mini). Cobrar R$0.10 por mensagem IA = margem de 90%. Cliente que gera 1.000 msgs/mes = R$100 extra. Escala: 500 clientes x 1.000 msgs = R$50k/mes adicional.

3. Automacoes extras cobradas

Plano base inclui cadencia padrao. Cadencias customizadas, playbooks avancados, condicoes com IA = add-on de R$197/mes. Alta margem, baixo custo de delivery.

4. Multi-seller como multiplicador

Cada seller adicional alem do plano = R$97/mes. Enterprise com 20 sellers = R$1.497 + 15 x R$97 = R$2.952/mes. O custo marginal de um seller e zero (e apenas uma role no sistema).

5. White-label para agencias

Agencias de growth e consultores de pagamento revendem o Shield com marca propria. Fee de R$497/mes por white-label + % sobre todos os clientes da agencia. Canal de distribuicao com custo de aquisicao zero.

6. Dados e benchmarks como produto

Com volume, o Shield tera o maior dataset de recovery do Brasil: taxas por nicho, metodo de pagamento, horario, valor. Esses benchmarks viram produto: relatorios pagos, API de inteligencia de mercado.

7. Integracao com gateways como moat

Cada gateway integrado aumenta switching cost. Cliente com Stripe + Pagar.me + Mercado Pago conectados nao sai facil. Partnerships com gateways podem gerar revenue share bidirecional.

RECORRENCIA
Por que o cliente nao sai

Recovery tools tem uma das maiores retencoes de qualquer vertical de SaaS B2B. O motivo e simples: o custo de nao usar e perder dinheiro todo dia.

Switching costs estruturais

  • Integracoes com gateways ja configuradas
  • Historico de leads e conversas no CRM
  • Playbooks otimizados ao longo do tempo
  • Sellers treinados na interface
  • Webhooks apontando para o Shield

Valor crescente no tempo

  • IA aprende padroes de falha do cliente
  • Templates otimizam com feedback de conversas
  • Dados historicos viram inteligencia de recovery
  • Benchmarks do setor disponives para comparacao
  • Cada mes de uso melhora a taxa de recuperacao
